# Plan 3a: MCP Server (sub-plan of Module 3)

**Complexity: ⚠️ Medium.** Small surface (one service module, one mount, no schema changes) but it introduces a new protocol dependency and lifespan wiring. Parent context: `.agent/plans/3.mcp-and-connectors.md`. Independently executable.

## Context

Module 2 built the tool registry (`backend/app/services/tools/`) with `execute_tool(conn, tenant_id, name, args, source_system=...)` as the **only** execution seam — it audits every call to `events` and refuses `safe=False` tools. This sub-plan exposes that registry to external MCP clients (Claude Code/claude.ai now; n8n custom nodes wrap the same server in Module 7) without duplicating any tool logic.

**User-locked decisions**: mounted in the existing FastAPI app (not a standalone process); Streamable HTTP transport at `/mcp`; static bearer token auth (`NEXUS_MCP_TOKEN`) until Module 6 replaces it with real auth; tenant identity from the existing `deps.get_tenant_id()` seam.

**Non-goals**: stdio transport (add later if a stdio-only client matters); OAuth/JWT on `/mcp` (M6); exposing MCP *resources*/*prompts* (tools only); write tools (M5); n8n consumption (M7).

## Key design decisions (locked in — do not re-litigate during build)

1. **Low-level `mcp.server.lowlevel.Server`, not FastMCP decorators.** FastMCP derives input schemas from Python signatures; our tools already carry JSON Schemas and a single dispatch seam. `list_tools` mirrors the registry (`registry.all_tools()` → `types.Tool(name, description, inputSchema)` — **no** `cache_control`, that's an Anthropic-API concern); `call_tool` dispatches every call through `execute_tool`. New tools registered in later modules appear over MCP with zero MCP-side changes.
2. **Dependency**: `mcp==1.28.1` (latest as of 2026-07-16, requires Python ≥3.10; we run 3.12). If the installed SDK's module paths differ from the notes below, adapt to the installed version rather than pinning older.
3. **Transport: Streamable HTTP, stateless, JSON responses.** `StreamableHTTPSessionManager(app=server, stateless=True, json_response=True)` mounted at `/mcp` via `app.mount(...)` on the ASGI handler. Stateless + JSON keeps tool calls simple request/response (no server-push needed for tools), works for n8n/HTTP clients, and makes tests plain httpx POSTs. The session manager's `run()` context manager is entered inside the existing FastAPI lifespan in `main.py`.
4. **Auth: bearer-token ASGI wrapper on the `/mcp` mount only.** Compares `Authorization: Bearer <token>` against `settings.nexus_mcp_token` with `hmac.compare_digest`. Unset token ⇒ every request 401s (fail closed; the rest of the API is unaffected). 401 is a plain HTTP response, not a JSON-RPC error — the request never reaches the MCP layer.
5. **`call_tool` handler**: opens `tenant_tx(get_tenant_id())`, runs `execute_tool(conn, tenant_id, name, args, source_system="mcp")`, returns `[types.TextContent(text=json.dumps({"summary": r.summary, "data": r.data}, default=str))]` with `isError` mirroring `ToolResult.is_error`. Never raises for a tool problem — same contract as chat. The `events` audit row therefore shows `source_system='mcp'`, distinguishing MCP-originated calls from chat.
6. **Config**: `nexus_mcp_token: str = ""` in `config.py`. No other settings.
7. **Observability**: `execute_tool` is already `@traceable(run_type="tool")`; wrap the `call_tool` handler with `@traceable(run_type="chain", name="mcp_call")` so MCP-originated traces group under one span like `chat_turn` does.

## Execution notes

- **Blocking user input**: none for Tasks 1–2 beyond the existing `NEXUS_APP_DB_URL` gate. Task 3 live check needs the user to pick/set `NEXUS_MCP_TOKEN` and have the Module 1 `.env` complete.
- Reuse: `tenant_tx` (`app/db.py`), `get_tenant_id` (`app/deps.py`), `traceable` (`app/llm.py`), `_require` skip pattern (`tests/conftest.py`), registry accessors (`services/tools/registry.py`).
- SDK notes (verify against installed 1.28.1): low-level server lives at `mcp.server.lowlevel.Server`; types at `mcp.types`; session manager at `mcp.server.streamable_http_manager.StreamableHTTPSessionManager`; `@server.list_tools()` / `@server.call_tool()` decorators; stateless mode tolerates per-request initialize-free calls but tests should still send `initialize` first to stay spec-shaped.
- Scope discipline: do not add MCP resources/prompts, stdio, or any tool definition changes. If a tool behaves wrong over MCP, the fix belongs in the tool/registry (shared with chat), not in MCP-specific branching.
