# Plan 1: Foundation Chat + Ingestion

**Complexity: ⚠️ Medium** (upper end). Not on CLAUDE.md's default-🔴 list (no MCP layer, approval gate, or harness). Large surface — a new migration set, the first FastAPI app, three external APIs, the first frontend — but every task is linear and independently validated. If execution stalls, natural split points are after Task 6 (ingestion backend done) and Task 8 (backend done, frontend remaining).

## Context

Module 0 delivered the canonical schema (hosted Supabase, ref `csiwxltfzodnlywuykdh`) with `documents`, `document_chunks` (vector(1024), HNSW cosine), immutable `events`, tenant RLS via `app.current_tenant_id()` (JWT claim → GUC `request.app.tenant_id` → deny), and a green 28-test pytest harness. Nothing runs yet: no FastAPI app, no frontend. Module 1 builds the first two user-facing interfaces per the PRD: **Chat** (threaded, SSE-streamed, basic RAG with citations) and **Ingestion** (drag-and-drop upload, 4-format parsing, chunk+embed pipeline, live status via Supabase Realtime).

**User-locked decisions**: chat includes basic RAG (plain pgvector cosine, top-k, no hybrid/rerank — that's Module 10); chat threads persist in new RLS-scoped core tables; all 4 formats parsed with lightweight parsers behind a swappable interface (Docling swap is Module 10); one plan, strictly ordered tasks.

**Non-goals**: MCP tools (M3), structured-data tools (M2), hybrid search/rerank/Docling (M10), auth UI (M6 — tenant from `NEXUS_TENANT_ID` env), webhook connectors (M3).

## Key design decisions (locked in — do not re-litigate during build)

1. **DB access: psycopg async pool + dedicated `nexus_app` role + `SET LOCAL` GUC — not PostgREST.** Verified: the `SUPABASE_DB_URL` role (`postgres`) has `BYPASSRLS`, so the backend must NOT connect as it — a new migration creates login role `nexus_app` (`nobypassrls`, `grant authenticated to nexus_app` so existing `to authenticated, anon` policies apply via role membership). Direct SQL is needed anyway for `<=>` vector search and batch chunk inserts. Every request runs in a transaction that does `select set_config('request.app.tenant_id', %s, true)`. Password is set via a one-time documented psql ops step (never committed); `.env` gains `NEXUS_APP_DB_URL` (pooler username `nexus_app.<project-ref>`). Tenant seam for Module 6: `deps.get_tenant_id()` returns `settings.nexus_tenant_id` today, JWT claim later.
2. **Chat persistence**: new core migration `chat_threads` (id, tenant_id, title, timestamps + `app.set_updated_at()` trigger) and `chat_messages` (id, tenant_id, thread_id FK cascade, `seq bigint generated always as identity` for stable ordering, `role check in ('user','assistant')`, `content jsonb` = Anthropic content-block array **verbatim** (forward-compatible with M2 `tool_use`/`tool_result` blocks, which live inside user/assistant messages), `citations jsonb default '[]'`, `metadata jsonb`, created_at). Indexes: `(tenant_id, updated_at desc)` on threads, `(thread_id, seq)` on messages. Four-policy RLS copied exactly from the `documents` pattern in `core_rls.sql`, incl. `force row level security`.
3. **Ingestion**: `POST /api/documents` (multipart) → insert row `status='uploaded'` → original bytes to private Storage bucket `documents` at `{tenant_id}/{document_id}/{filename}` → 202 → **FastAPI `BackgroundTasks`** runs `process_document()`: processing → parse → chunk → insert chunks (embedding NULL) → Voyage-embed in batches, UPDATE per batch → ready; any exception → failed + `error`. Each transition writes an `events` row (`source_system='ingestion'`, `entity_type='document'`, `event_type='document.uploaded|processing|ready|failed'`). Frontend observes the `documents` row via Realtime (`alter publication supabase_realtime add table public.documents`). No worker/queue — single tenant, human-paced uploads; CLAUDE.md scale discipline. Storage upload uses the service-role key (accepted, documented exception — Storage-only; per-tenant Storage RLS deferred to M6). Plus `GET /api/documents`, `GET /api/documents/{id}` (chunk previews), `DELETE /api/documents/{id}` (cascade + Storage object removal).
4. **Parser layer — swappable**: `DocumentParser` Protocol returning `ParsedDocument(text, metadata)`; registry keyed by extension: `pypdf` (.pdf), `python-docx` (.docx), BeautifulSoup `get_text` after dropping script/style (.html/.htm), UTF-8 decode (.md/.txt — raw Markdown embeds fine). Docling (M10) replaces registry entries only.
5. **Chunking**: pure function, ~1500 chars target, 200-char overlap, paragraph-boundary packing with sentence/whitespace hard-split for oversized paragraphs. `metadata={"char_start","char_end"}`. No tokenizer dep.
6. **Embeddings**: Voyage `voyage-3.5` (dim 1024 matches column), `embed_documents` batched ≤128 with `input_type="document"`, `embed_query` with `input_type="query"`, both `@traceable`. Write vectors as `'[...]'::vector` string literals (Module 0 test pattern). Assert dim==1024 before insert.
7. **Chat SSE**: `POST /api/chat/threads/{id}/messages` streams `text/event-stream`. Flow: persist user message → `embed_query` → retrieve top-8 → Anthropic call where **system is two blocks**: static persona/instructions with `cache_control: {"type":"ephemeral"}`, then per-turn retrieved context numbered `[1]..[8]` with filename headers (no cache_control — changes every turn). `messages` = full stored history verbatim + new message. Model: Claude Sonnet — **confirm current model id at build time, don't hardcode from memory**. On completion persist assistant message (content blocks, citations, usage metadata) + one `events` row (`source_system='chat'`, `event_type='chat.message.completed'`). SSE events: `start {thread_id,user_message_id}` → `citations {sources:[{n,document_id,filename,chunk_id,chunk_index,snippet}]}` → `text {delta}`* → `done {assistant_message_id,usage}`; `error {message}` on failure. Model instructed to cite `[n]` inline; frontend renders source list — no span-level grounding. Also: `POST/GET/DELETE /api/chat/threads`, `GET /api/chat/threads/{id}/messages`.
8. **Retrieval**: `@traceable`, on the tenant-scoped connection so **RLS does the tenant filtering** (proves the nexus_app+GUC path): `select c.*, d.filename from document_chunks c join documents d ... where c.embedding is not null order by c.embedding <=> %s::vector limit 8`. No threshold, no hybrid.
9. **Realtime auth (dev seam)**: anon Realtime client sees zero rows (fail closed, correct). Backend exposes `GET /api/auth/realtime-token` minting a 1h HS256 JWT with `SUPABASE_JWT_SECRET` + `app_metadata.tenant_id` (same pattern as test conftest; legacy JWT secret verified active). Frontend calls `supabase.realtime.setAuth(token)`. Module 6 replaces this endpoint with real Supabase Auth.
10. **Frontend**: Vite + React + TS + Tailwind + shadcn/ui + react-router-dom + @supabase/supabase-js in `frontend/`. Vite `server.proxy` `/api` → `localhost:8000` (no CORS friction; backend still adds localhost CORS). SSE consumed via `fetch` POST + `body.getReader()` frame parser (`EventSource` can't POST). Routes: `/` → Chat (default), `/ingestion` → Ingestion. No auth UI.
11. **LangSmith**: `app/llm.py` builds one `wrap_anthropic(anthropic.AsyncAnthropic())` client. `@traceable` on `process_document`, `embed_*`, `retrieve_chunks`, and the chat-turn orchestrator (trace shows retrieve → generate nested spans). Must no-op gracefully when `LANGSMITH_API_KEY` unset.
12. **Config**: `pydantic-settings` loading repo-root `.env`. New `.env.example` vars: `NEXUS_APP_DB_URL`, `ANTHROPIC_API_KEY`, `VOYAGE_API_KEY`, `LANGSMITH_TRACING`, `LANGSMITH_API_KEY`, `LANGSMITH_PROJECT=nexus`. requirements.txt additions: `fastapi uvicorn[standard] python-multipart pydantic-settings psycopg-pool anthropic voyageai langsmith pypdf python-docx beautifulsoup4 httpx`. Existing venv rule applies.

## Tasks

### Task 1 — Migrations: app role, chat tables, ingestion infra
1. `<ts>_app_role.sql`: idempotent `create role nexus_app login nobypassrls ...`; `grant authenticated to nexus_app`; comment. No password in SQL.
2. `<ts>_core_chat.sql`: `chat_threads` + `chat_messages` per D2, updated_at trigger, indexes, four-policy RLS for both (copy `documents` blocks in `core_rls.sql`).
3. `<ts>_ingestion_infra.sql`: `insert into storage.buckets (id,name,public) values ('documents','documents',false) on conflict do nothing`; `alter publication supabase_realtime add table public.documents`.
4. `supabase db push`.
5. **Blocking ops step** (document in README): user sets password via psql `alter role nexus_app with password '...'`, adds `NEXUS_APP_DB_URL` + new API keys to `.env`; update `.env.example`.

**Validation**: push clean; `backend/tests/test_chat_schema.py` — tables exist, 4 policies each, cross-tenant invisibility via existing `client_tenant_a/b` fixtures; `backend/tests/test_app_role.py` — psycopg as `nexus_app`: 0 rows without GUC, seeded rows with GUC, cross-tenant INSERT rejected (**proves the postgres-BYPASSRLS hole is closed**).

### Task 2 — Backend app skeleton
`app/` package: `config.py` (pydantic-settings), `db.py` (AsyncConnectionPool on `NEXUS_APP_DB_URL`; `tenant_conn` FastAPI dependency doing `set_config(..., true)` inside a transaction; `tenant_tx()` context manager for background tasks), `deps.get_tenant_id()`, `main.py` (lifespan pool open/close, CORS for localhost, `GET /healthz`). **Validation**: `backend/venv/Scripts/python -m uvicorn app.main:app` boots; `curl localhost:8000/healthz` → `{"status":"ok"}`; `tests/test_api_health.py` via httpx `ASGITransport`.
